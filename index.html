Proposed Schema for GIMP Extension API

Details

/repository Endpoints

/repository/metadata

HTTP Method: GET

  • ID: Endpoint-01
  • Description:Pull full list of extension metadata for repository.

/repository/extensions/{id}/{version}

HTTP Method: GET

  • ID:Endpoint-02
  • Description: Download extension package

/repository/search/{searchQuery}

HTTP Method: GET

  • ID:Endpoint-03
  • Description: Searches repository for list of extensions, filtered by search queries.

/extensions Endpoints

/extensions

HTTP Method: POST

  • ID:Endpoint-04
  • Description: Upload new extension

HTTP Method: GET

  • ID:Endpoint-05
  • Description: Returns package containing metadata details for single extension

HTTP Method: PUT

  • ID:Endpoint-06
  • Description: Updates the version of an extension

HTTP Method: DELETE

  • ID:Endpoint-07
  • Description: Deletes an extension and all its versions from the repository.

/extensions/{id}/flags

HTTP Method: POST

  • ID:Endpoint-08
  • Description: Flags an extension for potentially inappropriate content or poor quality / malicious software.

/extensions/santizeExtension

HTTP Method: PUT

  • ID:Endpoint-09
  • Description: Process for ensuring that extensions meet metadata and API guidelines before uploading it to the repository.

/auth Endpoints

/auth/register

HTTP Method: POST

  • ID:Endpoint-10
  • Description: Register new user.

/auth/login

HTTP Method: POST

  • ID:Endpoint-11
  • Description: Authenticates a user for a new session.
